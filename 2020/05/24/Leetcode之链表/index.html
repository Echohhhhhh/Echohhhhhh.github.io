1. 快慢指针

1.1. 找链表的中间节点

给定一个带有头结点 head 的非空单链表，返回链表的中间结点。

如果有两个中间结点，则返回第二个中间结点。

【分析】

  • 当链表中个数为奇数时，返回中间的节点。例如1->2->3->4->5，返回节点3
  • 当链表中个数为偶数时，返回右节点。例如1->2->3->4，中间的节点有2和3，返回右节点3
  • 当链表中个数为偶数时，返回左节点。例如1->2->3->4，中间的节点有2和3，返回右节点2

使用双指针，慢指针一次走一步，快指针一次走两步

1.2. 倒数第K个节点

输入一个链表，输出该链表中倒数第k个节点。尾节点是倒数第1个节点。

例如：给定一个链表: 1->2->3->4->5, 和 k = 2.
返回链表 4->5

【分析】
使用快慢指针，快指针先向前移动k个节点，慢指针指向head，然后快慢指针一起向前走，当快指针指向null时，此时慢指针指向倒数第k个节点，返回即可。
举例：1->3->9->7->6->12，k=3，首先fast和low都指向1，然后fast向前移动3步指向7，然后fast和low同时向前走，当fast指向null时，此时low指向7，返回即可。

1.3. 环形链表

https://leetcode-cn.com/problems/linked-list-cycle/
给定一个链表，判断链表中是否有环。

  • 使用set
    使用set保存节点的引用，而不是节点的值

  • 快慢指针
    快指针每次走2步，慢指针每次走1步，如果链表中有环，两者迟早会碰上，如果始终碰不上，则链表中没有环。

1.4. 环形链表II

给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。
说明：不允许修改给定的链表。
链接：https://leetcode-cn.com/problems/linked-list-cycle-ii

【分析】
https://leetcode-cn.com/problems/linked-list-cycle-ii/solution/wo-dui-di-er-jie-duan-de-li-jie-by-fei-er-10/

快指针每次走2步，慢指针每次走1步，如果链表中存在环，则一定会相遇。如上图所示，2指针在C点相遇，然后让一个指针退回到头结点A，一个指针继续待在C，然后向前走，两指针再次相遇的点就是入环的第一个节点。

1.5. 相交链表

编写一个程序，找到两个单链表相交的起始节点。
https://leetcode-cn.com/problems/intersection-of-two-linked-lists/

【分析】

A链表：1->2->3->8->7->9->11
B链表：10->6->8->7->9->11
相交节点为8，共同拥有部分长度c=4，A和B分别单独拥有的长度a=3，b=2
a+c不等于b+c，但如果2个节点同时遍历a+c+b,b+c+a，因为长度一样，则一定会同时结束。
使用指针A和指针B用来遍历链表1和链表2，当链表1遍历完之后，指针A指向链表2的头结点，开始遍历链表2。当链表2遍历完之后，指针B指向链表1的头结点，开始遍历链表1。

  • 如果链表1和链表2有交点，则A和B一定会在交点处遇见，此时返回即可。
  • 如果链表1和链表2不相交，则指针A会走到链表2的None，指针B会走到链表1的None，此时A==B，跳出循环，返回A，即返回None，表示没有交点。

2. 合并有序链表

2.1. 合并2个有序链表

将两个升序链表合并为一个新的 升序 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

示例：
输入：1->2->4, 1->3->4
输出：1->1->2->3->4->4

链接：https://leetcode-cn.com/problems/merge-two-sorted-lists

【分析】

新建一个链表来保存合并之后的值

l1和l2指向当前需要比较的2个节点

  • 如果l1小于l2，则将l1插入到新的链表中，然后将l1向后移一位，把新链表中的pre向后移一位。
  • 如果l2比较小，把l2插入到新链表中，然后将l2向后移一位，把新链表中的pre向后移一位。

跳出while循环时，肯定是有一个链表已经为空了，如果另一个链表不为空，则把这个链表直接加到新的链表中

  • 时间复杂度：O(n+m)，n和m表示2个链表的长度
  • 空间复杂度：O(1)

【分析二】

使用递归方法

2.2. 合并K个有序链表

合并 k 个排序链表，返回合并后的排序链表。请分析和描述算法的复杂度。

示例:
输入:
[
1->4->5,
1->3->4,
2->6
]
输出: 1->1->2->3->4->4->5->6

链接：https://leetcode-cn.com/problems/merge-k-sorted-lists

【分析一】

一共有K个有序链表，空链表先和链表1合并，得到链表a，链表a再和链表2合并得到链表b，。。。一直合并到链表K。但是这种方法超时

2个链表合并的时间复杂度是O(N)，其中N是2个链表节点数之和。
这里假设一个链表的节点个数为N，第一个合并时间复杂度为0，因为是和空链表合并，第二次合并复杂度为2N，第三个合并时间复杂度为3N，第K次合并时间复杂度为KN
即2N+3N+…+KN=(K+2)(K-1)N/2
近似是K*KN，这里把KN看成是所有链表的节点数N，则时间复杂度为O(KN)

【分析二】

分治合并
一共K个有序链表，先分组，每2个为一组，先把一组内的2个链表合并，变成K/2个链表，然后再分组再合并。
在合并2个有序链表时有2种方法：递归和迭代。
在本题中当使用迭代方式时，超时
使用递归方法不超时

假设K个链表一共有N个节点，每个链表有N/K个节点，2个链表合并的时间复杂度是o(N/K)，一个链表被合并logK次，一共有K个链表，则时间复杂度是O(NlogK)

4. 反转链表

4.1. 反转链表

【分析】

反转列表，原先的头结点变成尾节点，首先cur为头结点，pre为None，第一次循环时，先把cur的下一个节点保存成tmp，然后改变cur的next为None，这样就把头结点变成尾节点（next节点为空），然后cur变成pre，开始遍历下一个节点，即tmp，使用cur=tmp赋值。

4.2. K个一组反转链表

给你一个链表，每k个节点一组进行翻转，请返回翻转后的链表。k是一个正整数，它的值小于等于链表长度。
如果节点总数不是k的倍数，那么请将最后剩余的节点保持原有顺序

要求：

  1. 只能使用常数额外空间
  2. 不能只改变节点内部的值，而是要交换节点

示例：
给你这个链表：1->2->3->4->5
当 k = 2 时，应当返回: 2->1->4->3->5
当 k = 3 时，应当返回: 3->2->1->4->5

链接：https://leetcode-cn.com/problems/reverse-nodes-in-k-group

【分析】

需要把链表按照k个分成一组，使用head指向每组的头结点，然后向前移动K步，如果还没移动到k步就到达链表尾部，说明后面的这部分不够K个元素，就不用反转了，直接返回。如果够K个节点，则需要反转这K个节点。

reverse(self,head,tail)用来反转这K个元素，传入K个元素的头和尾，反转之后返回新的头和尾。

将这K个元素反转之后返回新的头和尾，在把这K个元素加到原先的链表上。
在反转之前，记录head的前一个节点，用pre表示，记录tail的后一个节点，用tail_next表示。反转之后，把新的head放在pre后面，把新的tail放在tail_next前面。则这K个元素就处理完了。接着处理下K个元素。需要移动head和pre，head移到下K个元素的头，即head=tail.next，pre移动到这K个元素的前一个节点，即pre=tail，然后继续while循环，tail此时指向pre，然后tail向前移动K个元素

  • 时间复杂度：O(n)遍历了整个链表，但每隔K个元素，会消耗O(K)的时间来进行反转
  • 空间复杂度：O(1)

5.1. 两两交换链表中的节点

给定一个链表，两两交换其中相邻的节点，并返回交换后的链表。

你不能只是单纯的改变节点内部的值，而是需要实际的进行节点交换。

https://leetcode-cn.com/problems/swap-nodes-in-pairs/

例如：给定 1->2->3->4, 你应该返回 2->1->4->3.

【分析】
在原始链表上添加一个头节点thread，将thread指向原始头结点head。每次交换顺序时，需要有3个指针：a：当前交换的左节点，b：当前交换的右节点，c：a的前一个节点。例如要交换1和2的顺序时，c=-1，a=1，b=2，交换后变成-1->2->1->3->4，要交换3和4的顺序时，c=1，a=3，b=4，交换后变成-1->2->1->4->3，然后返回thread的下一个节点，即新的头结点2。

5.2. 删除链表的倒数第N个节点

给定一个链表，删除链表的倒数第 n 个节点，并且返回链表的头结点。
https://leetcode-cn.com/problems/remove-nth-node-from-end-of-list/

给定一个链表: 1->2->3->4->5, 和 n = 2.
当删除了倒数第二个节点后，链表变为 1->2->3->5.

【分析】
这一题是”找链表中倒数第N个节点”的进阶版。首先使用快慢指针找到倒数第N个节点，同时还要删除，这里需要记录倒数第N个节点的前一个节点。本题通过添加一个头结点来实现。

编写代码，移除未排序链表中的重复节点。保留最开始出现的节点。

示例1:
输入：[1, 2, 3, 3, 2, 1]
输出：[1, 2, 3]

https://leetcode-cn.com/problems/remove-duplicate-node-lcci/submissions/

删除链表中的节点，根据经验，添加一个新的头结点

使用set来保存链表中不重复的值。当有一个节点在set中已经有了，则把这个节点删掉，即pre.next=cur.next，同时更新cur的值。当这个点在set中没有，pre和cur同时移向下一个节点

5.5. 删除中间节点

实现一种算法，删除单向链表中间的某个节点（即不是第一个或最后一个节点），假定你只能访问该节点。

输入：单向链表a->b->c->d->e->f中的节点c
结果：不返回任何数据，但该链表变为a->b->d->e->f

https://leetcode-cn.com/problems/delete-middle-node-lcci/

【分析】

第一次做误以为node就是链表的头结点，然后使用快慢指针找到链表的中间节点，并同时记录中间节点的前一个节点，来删除中间节点。

但是仔细读了题发现node并不是链表的头结点，而是需要删除的节点，这里并不知道链表的头结点，只知道被删除的这个节点。

1->2->3->4，假设需要节点2，这里只知道节点2，其余都不知道。

将节点3的值赋值给节点2，这样节点2的值变成3，然后在让这个节点指向节点4，最终变成1->3->4

6. 链表求和

6.1. 链表求和

给定两个用链表表示的整数，每个节点包含一个数位。
这些数位是反向存放的，也就是个位排在链表首部。
编写函数对这两个整数求和，并用链表形式返回结果。

示例：
输入：(7 -> 1 -> 6) + (5 -> 9 -> 2)，即617 + 295
输出：2 -> 1 -> 9，即912

https://leetcode-cn.com/problems/sum-lists-lcci/

【分析】

链表的第一个节点表示个位，即使2个链表长度不一样，也是从头结点+头结点开始。

  1. 先将对应位相加
  2. 再加上前一位的进位
  3. 根据当前位的和，模10，得到当前位的数，然后创建节点，并将这个节点加入到结果链表中
  4. 当前位的和除以10得到向下一位的进位

只有当2个链表中最长的那个链表遍历到末尾，并且没有前一位的进位时，才退出循环。

6.2. 进阶

进阶：假设这些数位是正向存放的，请再做一遍。

示例：
输入：(8 -> 6 -> 1 -> 7) + (2 -> 9 -> 5)，即8617 + 295
输出：8 -> 9 -> 1 -> 2，即8912

【分析】

个位数在最后一个节点，所以从最后一个节点开始相加，但是链表无法刚开始就取到最后一位，这里利用栈来存储链表中的值。栈先进后出，现将高位的数字放到栈中，个数为最后放入栈中，在相加的时候从栈中取数据，先取出个位数，然后相加，再加上进位。将得到的和模10得到当前位的值并创建节点。这里需要注意再将节点加入到结果链表中，个位的节点在最后一个，其余的节点都在它前面。

7. 回文链表

判断一个链表是否是回文链表

示例 1：
输入： 1->2
输出： false
示例 2：
输入： 1->2->2->1
输出： true

https://leetcode-cn.com/problems/palindrome-linked-list-lcci/

【分析】

回文链表即链表的后半部分反转之后和前半部分相等。所以判断一个链表是否是回文链表分为3步：

  1. 找到链表的中间节点（左中位数）
  2. 反转后半部分
  3. 比较前后是否相等

8. 重排链表

【分析】

分为3步：

  1. 找出链表的中位数
  2. 反转后半部分
  3. 将前后两部分交叉拼接

这道题和回文链表的前2步相似

这里找的是左中位数，但是反转的时候是从右中位数开始反转。
例如1->2->3->4，找到的左中位数是2，对3->4进行反转，然后将2的next置为None
例如1->2->3->4->5，找到的中位数是3，对4->5进行反转，然后将3的next置为None。
反转之后，得到前半部分为1->2->3->None，后半部分为5->4->None，然后将这2部分交叉拼接

9. 总结

  1. 使用快慢指针
    链表中间节点，链表倒数K个元素，环形链表
  2. 添加新的头结点
    当在链表中删除元素时，一般需要添加一个新的头节点，next指向原来的头结点，注意返回的时候返回原先的头结点，而不是新建的。
